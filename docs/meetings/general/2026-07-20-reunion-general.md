# Temas tratados

## 1. Organización del repositorio

Se acordó mantener, en la medida de lo posible, la estructura del repositorio de GitHub previamente definida para el proyecto, con el objetivo de conservar una organización homogénea entre todos los equipos.

---

## 2. Integración con Data Science

El equipo de Data Science presentó la estrategia definida para el manejo de los datos.

Se acordó que:

- El dataset será almacenado y administrado utilizando **MySQL**.
- La información será entregada al equipo de Backend en formato **JSON** para facilitar la integración.
- No se realizará migración entre motores de bases de datos durante el desarrollo del MVP.

---

## 3. Presentación del plan de trabajo de Data Science

El equipo de Data Science presentó:

- El plan de trabajo correspondiente a la semana actual.
- La estructura de actividades planificadas para las siguientes semanas.
- La estrategia de desarrollo del dataset y las fases previstas para el área de Ciencia de Datos.

---

## 4. Coordinación entre Backend y Data Science

Se identificó la necesidad de realizar una reunión de trabajo con el equipo de Backend con el propósito de conocer el plan de trabajo de Backend y alinear las actividades de ambos equipos.

Durante esta reunión se buscará:

- Presentar o crear el plan de trabajo del equipo Backend.
- Conocer el cronograma de actividades y entregables.
- Identificar los puntos de integración entre Backend y Data Science.
- Definir los tiempos de entrega de la información.
- Alinear expectativas para evitar dependencias o retrasos durante el desarrollo del MVP.
